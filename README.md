# Robot RSI + MA (MQL5)

Торговый советник для **MetaTrader 5**, торгующий на пересечении двух экспоненциальных скользящих средних (EMA 5 и EMA 12) с подтверждением от индикатора RSI.

## Описание

Робот анализирует текущий бар и открывает позицию **только один раз на новом баре**, если выполнены условия входа.  
Сигнал генерируется при одновременном выполнении:

- **Для BUY**:  
  - EMA5 пересекает EMA12 снизу вверх  
  - RSI пересекает уровень 50 снизу вверх (значение RSI[0] > 50, а RSI[1] ≤ 50)

- **Для SELL**:  
  - EMA5 пересекает EMA12 сверху вниз  
  - RSI пересекает уровень 50 сверху вниз (значение RSI[0] < 50, а RSI[1] ≥ 50)

После открытия позиции устанавливаются Stop Loss и Take Profit (через модификацию ордера) с проверкой минимального допустимого расстояния от текущей цены.

## Особенности

- Открытие только **одной позиции** за один бар (флаг `attemptOnCurrentBar`).
- **Отсутствие одновременных противоположных позиций** – проверка существующей позиции по магическому номеру и символу.
- **Обход ошибок при установке SL/TP**:
  - Ордер сначала открывается без стоп-лосса и тейк-профита.
  - Затем выполняется отдельный запрос на модификацию (TRADE_ACTION_SLTP).
  - Если заданные уровни меньше минимально разрешённых брокером, они автоматически поднимаются до `SYMBOL_TRADE_STOPS_LEVEL`.
- Корректная нормализация цен и объёмов лота (с учётом `SYMBOL_VOLUME_MIN/MAX/STEP`).

## Входные параметры (Input)

| Параметр       | Тип    | Значение по умолчанию | Описание                                    |
|----------------|--------|-----------------------|---------------------------------------------|
| LotSize        | double | 0.01                  | Фиксированный объём позиции                 |
| StopLoss       | int    | 50                    | Стоп-лосс в пунктах                         |
| TakeProfit     | int    | 100                   | Тейк-профит в пунктах                       |
| RSIPeriod      | int    | 14                    | Период RSI                                  |
| RSILevel       | int    | 50                    | Уровень RSI для подтверждения сигнала       |

> **Примечание:** Периоды скользящих средних жёстко заданы в коде: быстрая EMA = 5, медленная EMA = 12.

## Логика работы (OnTick)

1. Определяется начало нового бара (сравнение `iTime(_Symbol, _Period, 0)`).
2. Если на текущем баре уже была попытка открытия (`attemptOnCurrentBar` = true) **или** уже есть открытая позиция – выход.
3. Получаются значения индикаторов (по 2 последних элемента).
4. Проверяются условия входа.
5. При совпадении условий вызывается `OpenBuyOrder()` или `OpenSellOrder()`.

## Функции

- `PositionExists()` – проверяет наличие позиции с магическим номером `64385` по текущему символу.
- `ValidateLotSize()` – корректирует лот в соответствии с допустимыми значениями символа.
- `GetValidStopLevel()` – возвращает максимальное из запрошенного значения и минимально разрешённого брокером.
- `NormalizePrice()` – округляет цену до количества знаков символа.
- `OpenBuyOrder()` / `OpenSellOrder()` – открытие ордера (без стопов) и последующая модификация с установкой SL/TP.

## Требования

- **MetaTrader 5** (любая версия)
- Разрешённая торговля на счёте (реальный или демо)
- Наличие инструмента с котировками на текущем графике

## Установка

1. Скачайте файл `robot_rsi_min.mq5`.
2. В терминале MetaTrader 5 откройте **File → Open Data Folder** → перейдите в папку `MQL5/Experts/`.
3. Поместите файл в эту папку.
4. В редакторе MetaEditor (**MetaEditor → File → Open**) откройте файл и нажмите **Compile** (F7) – должен получиться файл `.ex5`.
5. В терминале прикрепите советник к графику (перетащите из панели Navigator).
6. Установите входные параметры при необходимости, разрешите **Allow Algo Trading**.

## Замечания

- Робот **не использует управление капиталом** – фиксированный лот.
- **Не применяет трейлинг-стоп** или другие методы сопровождения позиций.
- **Не обрабатывает** повторные сигналы в рамках одного бара (только одна попытка).
- Уровни SL/TP модифицируются **после открытия позиции**, что позволяет обойти ограничения брокеров, запрещающих установку слишком близких стопов при отправке рыночного ордера.
- Если модификация не удалась – позиция остаётся без стоп-лосса и тейк-профита. Это следует учитывать при торговле.
